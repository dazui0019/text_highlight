# Polarion Step Assistant

一个面向 Polarion 测试执行页面的 Chrome 扩展，用来切换工步并展示当前工步相对前序工步发生变化的信号。

## 使用方法

1. 打开 Chrome，进入 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择当前目录 `d:\File\AI\Vibe\text_highlight`
5. 打开 Polarion 工步执行页面，右侧会自动出现“Polarion 工步助手”浮窗

## 功能

- 在 Polarion 页面右侧显示工步助手浮窗
- 支持上一条、下一条和下拉选择工步
- 切换工步时自动滚动到页面对应位置
- 自动解析 `Step Description`
- 按正则提取 `变量名=值` 形式的信号
- 显示当前工步相对前序工步发生变化的信号
- 支持在插件弹窗中配置信号提取正则，页面浮窗不显示正则输入框
- 页面异步刷新后会自动重新扫描工步

## 默认正则

```regex
([A-Za-z_][A-Za-z0-9_]*(?:\s*\/\s*[A-Za-z_][A-Za-z0-9_]*)*)\s*=+\s*((?:0x[0-9A-Fa-f]+|\d+)(?:\s*\/\s*(?:0x[0-9A-Fa-f]+|\d+))*)
```

说明：

- 第 1 组表示信号名
- 第 2 组表示信号值
- 像 `LHTrunLight/RHTrunLight=0x1` 会拆成两个信号，值都为 `0x1`
- 像 `DaytimeLight=0x1/0x2/0x3点灯指令` 会只提取 `0x1/0x2/0x3`

## 说明

- 插件代码更新后，需要到 `chrome://extensions/` 点一次“重新加载”
- 如果你的 Polarion 页面 DOM 结构和当前截图差异较大，工步识别规则可能还需要继续微调
- `chrome://`、扩展商店等受限页面无法注入内容脚本
